# Talent Acquisition Analytics Project

A recruitment funnel analytics project — built to demonstrate data-driven
sourcing and hiring analysis skills for a Talent Acquisition role.

## What's inside

| File | What it is |
|---|---|
| `candidates.csv` | Synthetic dataset — 320 candidate records (source, role, department, funnel dates, outcome) |
| `recruitment_analysis.sql` | Schema + 8 analytical SQL queries (funnel conversion, source ROI, time-to-hire, rejection reasons, monthly trend, role×source combos) |
| `dashboard.html` | Interactive recruitment dashboard — open directly in any browser, no install needed |

## How to use this for your resume

**Resume line (pick one):**
- "Built a recruitment funnel analytics dashboard tracking 320+ candidates across 6 sourcing channels, identifying referral programs as the highest-converting channel and reducing analysis time using SQL + interactive dashboards."
- "Designed SQL queries and a Power BI-style dashboard to analyze source-wise conversion, time-to-hire, and drop-off reasons for a simulated hiring pipeline."

**Where to host it:**
1. Put all 3 files in a GitHub repo (e.g. `talent-acquisition-analytics`)
2. Enable **GitHub Pages** on the repo so `dashboard.html` is viewable via a live link — this gives you a clickable project link for your resume, not just a screenshot
3. Add a short repo description + this README

## Talking points for interviews

Be ready to explain, in your own words:

1. **Why these metrics matter** — conversion rate by source tells you where to invest sourcing budget; time-to-hire tells you which departments have bottlenecks; rejection reasons tell you whether the problem is sourcing quality, interview process, or compensation.
2. **What you'd do with the referral insight** — referral converts best in this dataset, so you'd recommend expanding the employee referral program / incentive.
3. **What "offer decline rate" tells a recruiter** — high decline rate after offer usually points to compensation benchmarking issues or slow process (candidates accept competing offers while waiting).
4. **How you'd extend it with real company data** — swap the CSV for your actual ATS export, same SQL queries and dashboard logic apply directly.

## Making it your own (recommended before adding to resume)

- Regenerate the dataset with different numbers/seed so it doesn't look identical to anyone else's version of this project
- If you have real (anonymized) campus placement or internship recruitment numbers, swap those in — real numbers, even small scale, are always stronger than synthetic ones
- Add 2-3 lines of your own written "insights" summary at the top of the dashboard — this is what actually gets discussed in interviews

# Sourcing Toolkit — Job Descriptions + Boolean Search Strings

A ready-to-use sourcing kit for 5 common roles. Each entry has a short JD and
Boolean search strings for LinkedIn Recruiter, LinkedIn (free), and Naukri —
the actual mechanics of finding candidates, not just screening them.

---

## 1. Software Engineer (Backend)

**Job Description (short form)**
Build and maintain backend services handling core product features. Own API
design, database schema decisions, and performance under load. Collaborate
with product and frontend teams on end-to-end delivery.

**Must-haves:** 1-3 yrs experience, one backend language (Java/Python/Node),
REST API design, SQL

**LinkedIn Boolean:**
```
("backend developer" OR "backend engineer" OR "software engineer") AND (Java OR Python OR "Node.js") AND ("REST API" OR microservices) NOT "frontend only"
```

**Naukri Boolean:**
```
("backend developer" OR "software developer") AND (Java OR Python OR Node) AND (SQL OR MySQL OR PostgreSQL) AND experience:(1-3)
```

**X-ray (Google) search:**
```
site:linkedin.com/in ("backend engineer" OR "backend developer") Java Python "REST API" -recruiter
```

## 2. Data Analyst

**Job Description (short form)**
Turn raw business data into decisions. Own dashboards, ad-hoc analysis, and
recurring reporting for stakeholders across sales, marketing, and ops.

**Must-haves:** SQL, Excel/Power BI or Tableau, basic statistics, 0-2 yrs experience

**LinkedIn Boolean:**
```
("data analyst" OR "business analyst") AND (SQL AND ("Power BI" OR Tableau)) AND (Excel OR "data visualization") NOT "senior" NOT "lead"
```

**Naukri Boolean:**
```
("data analyst") AND (SQL) AND ("power bi" OR tableau OR excel) AND experience:(0-2)
```

## 3. Sales Executive

**Job Description (short form)**
Drive new customer acquisition through outbound outreach, product demos, and
deal closure. Own a monthly quota and pipeline hygiene in CRM.

**Must-haves:** B2B/B2C sales experience, CRM familiarity, target-driven track record

**LinkedIn Boolean:**
```
("sales executive" OR "business development executive" OR "BDE") AND (B2B OR B2C) AND (quota OR target OR "sales target") NOT "sales manager" NOT "regional head"
```

**Naukri Boolean:**
```
("sales executive" OR "business development") AND (CRM OR Salesforce) AND experience:(1-3)
```

**Job Description (short form)**
Support the full employee lifecycle — onboarding, HRMS data management,
policy queries, and coordination with TA on interview scheduling.

**Must-haves:** HR fundamentals, HRMS tool exposure, strong communication, 0-2 yrs experience

## How to use these in an interview

If asked "how do you source candidates for a role you've never hired for
before," walk through this exact process:

1. **Decode the JD into 3-5 non-negotiable keywords** (skills, tools, years of experience) — not every buzzword in the JD, just what actually predicts success.
2. **Layer Boolean logic**: `AND` for must-haves, `OR` for synonyms of the same skill, `NOT` to exclude seniority levels or adjacent roles that keep polluting results.
3. **Test on 2-3 platforms** (LinkedIn, Naukri, X-ray via Google) since candidate pools don't overlap perfectly across platforms.
4. **Iterate based on result count** — too few results, drop a constraint; too many irrelevant ones, add a NOT clause or tighten an AND.

## Making this your own

- Swap in the actual JD language your target company uses — recruiters can usually tell a templated JD from a role-specific one
- If you've done any real sourcing (even informally, for a college club or a friend's startup), replace one of these 5 with that real example
